# Architecture

## Общая схема

Сервис состоит из пяти основных компонентов:

1. `FastAPI` принимает HTTP-запросы.
2. `PostgreSQL` хранит пользователей, устройства, показания и результаты анализа.
3. `Redis` используется Celery как брокер и backend результатов.
4. `Celery worker` выполняет аналитические задачи в фоне.
5. `Flower` показывает состояние очередей и воркеров.

## Слои приложения

- `app/api` - маршруты API
- `app/services` - прикладная логика
- `app/db` - модели и подключение к БД
- `app/models` - Pydantic-схемы
- `app/tasks` - Celery и фоновые задачи

## Поток данных

1. Клиент создает пользователя и устройство.
2. Клиент отправляет показания `x`, `y`, `z`.
3. Показания сохраняются в таблицу `readings` с временной меткой.
4. Клиент инициирует анализ устройства или пользователя.
5. API возвращает `task_id`.
6. Celery считает метрики и сохраняет результат.
7. Клиент забирает результат по `task_id`.

## Таблицы

- `users`
- `devices`
- `readings`
- `analysis_results`

## Аналитика

Для каждой оси считаются:

- `min`
- `max`
- `sum`
- `count`
- `median`
