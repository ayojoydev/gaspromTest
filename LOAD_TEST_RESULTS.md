# Load Test Results

Документ фиксирует результаты нагрузочного тестирования, приложенные в репозитории.

## Инструмент

- Locust 2.17
- сценарий: [tests/locustfile.py](tests/locustfile.py)

## Контур

- API
- PostgreSQL
- Redis
- Celery worker
- Docker Compose

## Зафиксированные сценарии

### 50 пользователей

- продолжительность: 10 минут
- общий объем: 15 240 запросов
- суммарный RPS: 25.4
- среднее время ответа: 148 ms

### 100 пользователей

- продолжительность: 15 минут
- общий объем: 45 800 запросов
- пиковый RPS: 51.2
- среднее время ответа: 165 ms

### 200 пользователей

- продолжительность: 10 минут
- общий объем: 78 500 запросов
- пиковый RPS: 130.8
- среднее время ответа: 215 ms

## Наблюдения

- операции записи показаний были самыми быстрыми
- пакетная запись работала стабильно под большей нагрузкой
- аналитические endpoints отвечали заметно медленнее, чем прием данных
- при росте числа пользователей увеличивалась доля ошибок на аналитике

## Краткий вывод

По сохраненной сводке сервис уверенно выдерживает умеренную нагрузку на операции записи. Аналитика остается более тяжелой частью системы и первой требует оптимизации при росте параллелизма.

## Примечание

Если результаты будут использоваться в финальной сдаче, стоит повторно прогнать Locust после всех правок кода и обновить этот файл уже по свежим измерениям.
